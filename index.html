WHAT IS PROJECT TANGO?

Brief Introduction

Project Tango technology gives mobile devices the ability to navigate the physical world similar to how we do as humans. Also it brings a new kind of spatial perception by adding advanced computer vision, image processing, and special vision sensors. With the devices, users can experience to explore physical space around, including precise navigation without GPS, windows into virtual 3D worlds, measurement and scanning of spaces, and games that know where they are in the room and whatâ€™s around them.

Devices

Project Tango needs specific device to develop. The device has the sensors to be able to know where it is and how it moves through space. For now, all developers can buy the develop kit on Google shop.



Platform

Users can experience a leading-edge platform that brings devices a new level of awareness through area learning and real-time 3D motion and depth sensing. Project Tango can only be developed on Android platform.


Moblie apps

More and more applications released on Google shop. Indoor navigation, new home decorating, real life dimention, VR games... All of these applications fully use the new functions of Project Tango.


APIs development

Project Tango offers APIs in C and Java, and an SDK for Unity. For example, Java API: Package com.google.atap.tangoservice, and Java Support Library API: Package com.projecttango.tangosupport.

Project Tango Tablet Development Kit hardware

Sensors combination:

  • Motion tracking camera
  • 3D depth sensing

Project Tango uses a wide-angle camera, also called "Fisheye" lens. For my understanding, motion tracking camera provides the function of capture images and 3D tracking. It recreates a virtual room that fits perfectly to the filmed footage to assign the movements of a real camera to a digital one. In a bright enough environment, camera can catch enough points close to or far away from the camera. According to the points from images, calculate the relaition of the moving points.

3D depth sensors must have two elements: IR(Infra-Red) projector and IR camera. Some depth sensors have RGB(Red green blue) camera, and some don't have. For Project Tango tablet, it has RGB camera. All of these elements can be seen on the tablet diagram.

The IR projector projects a pattern of IR light which falls on objects around it like a sea of dots. And these dots can only been seen by IR camera. Then finally RGB camera capture the images and sends the video feed of disorted dot pattern into depth sensor processors. The processor works out depth from the displacement of the dots. This depth result map can be read from the depth sensor by the tablet.

The following sensors already build in commen mobile devices.



Ambient light sensor:

It is used for background light brightness automatically controller. It can benifits the portable device, especially for mobile devices' battery's life.



Barometer sensor:

It is high-accuracy chip which is used for temperature and barometric pressure detect and meassurement.



Gyroscope:

It is a spinning wheel or disc in which the axis of rotation is free to assume any orientation by itself.

CONCEPT

  • Motion Tracking

  • Area Learning

  • Depth Perception

The Project Tango APIs provide the position and orientation of the user's device in full six degrees of freedom; this combination of position and orientation is referred to as the deviceâ€™s pose. The APIs support two ways to get pose data: callbacks to get the most recent pose updates, and functions to get a pose estimate at a specific time.

Motion Tracking is great by itself if you need to know a device's orientation and relative position, but it does have limitations:
(1) It does not give the device the ability to understand the actual area around it.
(2) It does not "remember" previous sessions. Every time you start a new Motion Tracking session, the tracking starts over and reports its position relative to its most recent starting position.
(3) Over long distances and periods of time the accumulation of small errors can cause measurements to "drift," leading to larger errors in absolute position.

Area learning gives the device the ability to see and remember the key visual features of a physical space-- the edges, conners, other unique features. So it can recoginize the area again when is needed to recall. And this is working along with Motion Tracking. Without any cloud services, the device can quickly call out the mathematical description, which is called ADF(Area Description File). It is a memory file. Because a place can be changed, so it has more than one ADFs is more reliable.

Area learning can improve the accurancy of the motion tracking, even over time. If the device recognize the place which it has seen before, it will correct and adjust its path to be more consistent with its previous ovservations. It is called Drift Corrections. As image below.

Depth perception let the device understand the distance from object to object in the real world.

By combining Depth Perception with Motion Tracking, the device can measure distances between points in an area that aren't in the same frame. The device (for now) is designed work best indoor at moderate distances(0.5-4 meters). it may not be ideal for close-range object scanning or gesture detection.

There is a API provides a function to get depth data in the form of a point cloud. This format gives (x, y, z) coordinates for as many points in the scene as possible to calculate. Each dimension is a floating point value recording the position of each point in meters in the coordinate frame of the camera.

}

The normal motion tracking system lifecycle consists of three states: TangoPoseData.POSE_INITIALIZING, TangoPoseData.POSE_VALID, and TangoPoseData.POSE_INVALID.
In order to use motion tracking, your TangoConfig must have KEY_BOOLEAN_MOTIONTRACKING set to true.

There are two configuration options are relevant to area learning: Loading area descriptions and Learning mode
For Loading area descriptions, to load an area description file, the TangoConfig must have KEY_STRING_AREADESCRIPTION set to the UUID of the ADF you wish to load. When the TangoService starts, it loads this area description.
For Learning mode, In order to use area learning, your TangoConfig must have KEY_BOOLEAN_LEARNINGMODE set to true. In the default TangoConfig, KEY_BOOLEAN_LEARNINGMODE is set to false.
